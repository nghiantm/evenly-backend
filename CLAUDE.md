# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Git Workflow

**Never push directly to `main`.** The branch is protected — all changes must go through a pull request, and the `test` CI check must pass before merging. Always push to a feature branch and open a PR.

## Commands

```bash
# Install dependencies
pip install -e ".[dev]"

# Start dev server
uvicorn app.main:app --reload

# Run all tests (uses in-memory SQLite, no real DB needed)
pytest

# Run a single test file
pytest tests/test_groups.py

# Run a single test
pytest tests/test_groups.py::test_create_group

# Run database migrations
DATABASE_URL=postgresql+asyncpg://user:pass@host:port/dbname alembic upgrade head
```

Environment: copy `.env.example` to `.env` and fill in `DATABASE_URL`, `CLERK_JWKS_URL`, `CLERK_ISSUER`.

## Architecture

**Stack**: FastAPI + SQLAlchemy 2.x async + asyncpg (PostgreSQL) + Pydantic v2 + Clerk JWT auth

**Layers**:
- `app/routers/` — HTTP handlers, minimal logic, delegate to services
- `app/services/` — All business logic (expenses, balances, groups, settlements, users)
- `app/models/` — SQLAlchemy ORM entities
- `app/schemas/` — Pydantic request/response models with `from_orm_model()` helpers
- `app/core/security.py` — Clerk JWT verification (RS256/JWKS with TTL cache), `get_current_user` FastAPI dependency
- `app/core/config.py` — Settings loaded from env vars

**Balance source of truth — `GroupTransfer`**: Every expense and settlement produces `GroupTransfer` rows. A transfer `(from_user → to_user, amount)` means `from_user` owes `to_user`. Balance = `SUM(transfers where to_user=X) - SUM(transfers where from_user=X)`. Settlements create a reverse transfer to cancel debt. The entire balance state can be rebuilt from scratch via `balances.recalculate`.

**Expense creation flow**: Expense → ExpensePayments (who paid) + ExpenseSplits (who owes) → GroupTransfers generated by crossing payments × splits proportionally.

**Soft deletes**: Expenses use `deleted_at`; Groups use `archived_at`. Filter these in queries.

**Testing**: `tests/conftest.py` sets up an in-memory SQLite async engine and mocks `get_current_user`. New endpoint tests should follow the same fixture pattern.

## API Endpoints

All endpoints except `GET /healthz` require `Authorization: Bearer <clerk_jwt>`.

---

### Health

#### `GET /healthz`
No auth required.

**Response**
```json
{ "status": "ok" }
```

---

### Users

#### `GET /users/me`
Returns the authenticated user's profile. Auto-creates/updates the local user record from the Clerk JWT claims on first call.

**Response**
```json
{
  "userId": "uuid",
  "externalSubject": "user_2abc...",
  "email": "alice@example.com",
  "displayName": "Alice",
  "emailVerified": true,
  "status": "active",
  "defaultCurrency": "USD",
  "avatarUrl": "https://..."
}
```

---

#### `PATCH /users/me`
Update profile fields. All fields optional.

**Request**
```json
{
  "displayName": "Alice Smith",
  "defaultCurrency": "EUR",
  "avatarUrl": "https://..."
}
```

**Response** — same shape as `GET /users/me`

---

#### `GET /users/search?q=alice&limit=10`
Search users by email or display name. `q` required (min length 1), `limit` 1–50 (default 10).

**Response**
```json
[
  {
    "userId": "uuid",
    "email": "alice@example.com",
    "displayName": "Alice",
    "avatarUrl": "https://..."
  }
]
```

---

#### `GET /users/{user_id}`
Get any user by UUID.

**Response** — same shape as `GET /users/me`

### Groups

#### `POST /groups` — 201
Creator becomes OWNER.

**Request**
```json
{
  "name": "Apartment",
  "defaultCurrency": "USD"
}
```

**Response**
```json
{
  "groupId": "uuid",
  "name": "Apartment",
  "defaultCurrency": "USD",
  "simplifyDebts": false,
  "createdAt": "2024-01-01T00:00:00",
  "updatedAt": "2024-01-01T00:00:00",
  "archivedAt": null,
  "memberCount": 1,
  "myRole": "OWNER"
}
```

---

#### `GET /groups`
List all groups the current user belongs to.

**Response** — array of group objects (same shape as above)

---

#### `GET /groups/{group_id}`
Detailed group info including member list and activity counts. Requester must be an active member.

**Response** — extends the group object with:
```json
{
  "...group fields...",
  "members": [
    {
      "userId": "uuid",
      "email": "alice@example.com",
      "displayName": "Alice",
      "role": "OWNER",
      "joinedAt": "2024-01-01T00:00:00"
    }
  ],
  "recentExpenseCount": 12,
  "recentSettlementCount": 3
}
```

---

#### `PATCH /groups/{group_id}`
Update group settings. Requires OWNER or ADMIN role. All fields optional.

**Request**
```json
{
  "name": "Beach House",
  "defaultCurrency": "EUR",
  "simplifyDebts": true
}
```

**Response** — group object

---

#### `DELETE /groups/{group_id}` — 204
Archives the group (sets `archivedAt`). Requires OWNER role. No response body.

---

#### `POST /groups/{group_id}/members` — 201
Add a user to the group. Requires OWNER or ADMIN.

**Request**
```json
{
  "userId": "uuid",
  "role": "MEMBER"
}
```

**Response**
```json
{
  "userId": "uuid",
  "email": "bob@example.com",
  "displayName": "Bob",
  "role": "MEMBER",
  "joinedAt": "2024-01-15T00:00:00"
}
```

---

#### `DELETE /groups/{group_id}/members/{user_id}` — 204
Remove a member. Users can remove themselves; OWNER/ADMIN can remove others. No response body.

Only the expense creator or OWNER/ADMIN can update. All fields optional. If `paidBy`/`splits` are provided they fully replace the existing ones (and GroupTransfers are regenerated).

**Request** — same fields as create, all optional

```json
  "paidBy": [ { "userId": "uuid-alice", "amount": 95.00 } ],

    { "userId": "uuid-alice", "amountOwed": 31.67, "splitType": "EXACT" },
    { "userId": "uuid-bob",   "amountOwed": 31.67, "splitType": "EXACT" },
    { "userId": "uuid-carol", "amountOwed": 31.66, "splitType": "EXACT" }
  ]
}
```

**Response** — updated expense object

#### `DELETE /groups/{group_id}/expenses/{expense_id}` — 204
Soft-delete (sets `deleted_at`). Only creator or OWNER/ADMIN. No response body.

### Balances

#### `GET /groups/{group_id}/balances?currency=USD`
Group-wide balance summary. `currency` defaults to `USD`.

**Response**
```json
{
  "groupId": "uuid",
  "currency": "USD",
  "userBalances": [
    { "userId": "uuid-alice", "displayName": "Alice", "netAmount": 60.0 },
    { "userId": "uuid-bob",   "displayName": "Bob",   "netAmount": -30.0 },
    { "userId": "uuid-carol", "displayName": "Carol", "netAmount": -30.0 }
  ],
  "pairwiseDebts": [
    { "fromUserId": "uuid-bob",   "toUserId": "uuid-alice", "amount": 30.0, "currency": "USD" },
    { "fromUserId": "uuid-carol", "toUserId": "uuid-alice", "amount": 30.0, "currency": "USD" }
  ]
}
```
Positive `netAmount` = owed money; negative = owes money.

---

#### `GET /groups/{group_id}/balances/recalculate` — 204
Rebuilds all `GroupTransfer` rows from scratch by replaying every non-deleted expense and settlement. Requires OWNER or ADMIN. No response body.

---

#### `GET /users/me/balances`
Current user's aggregate balance across all groups.

**Response**
```json
{
  "youOwe": 45.00,
  "youAreOwed": 120.00,
  "netByGroup": [
    { "groupId": "uuid", "groupName": "Apartment", "currency": "USD", "netAmount": 75.00 }
  ],
  "netByUser": [
    { "userId": "uuid-bob", "displayName": "Bob", "currency": "USD", "netAmount": -30.00 }
  ]
}
```

### Settlements

#### `POST /groups/{group_id}/settlements` — 201
Record that `fromUserId` paid `toUserId`. Creates a reverse `GroupTransfer` to cancel the debt.

**Request**
```json
{
  "fromUserId": "uuid-bob",
  "toUserId": "uuid-alice",
  "currency": "USD",
  "amount": 30.00,
  "settlementDate": "2024-03-20",
  "note": "Venmo'd you"
}
```

**Response**
```json
{
  "settlementId": "uuid",
  "groupId": "uuid",
  "fromUserId": "uuid-bob",
  "toUserId": "uuid-alice",
  "currency": "USD",
  "amount": 30.0,
  "settlementDate": "2024-03-20",
  "note": "Venmo'd you",
  "createdBy": "uuid-bob",
  "createdAt": "2024-03-20T15:00:00"
}
```

---

#### `GET /groups/{group_id}/settlements`
**Response**
```json
{
  "settlements": [ /* array of settlement objects */ ],
  "total": 5
}
```

---

#### `GET /groups/{group_id}/settlements/{settlement_id}`
**Response** — single settlement object

---

#### `DELETE /groups/{group_id}/settlements/{settlement_id}` — 204
Deletes the settlement and reverses its `GroupTransfer`. Only creator or OWNER/ADMIN. No response body.
